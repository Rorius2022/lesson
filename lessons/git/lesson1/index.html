Тема: Ознайомлення з Git

  • Що таке Git?

    Git — це інструмент для керування версіями, який відстежує зміни в вашому коді протягом часу. Використовується багатьма розробниками програмного забезпечення та дозволяє ефективно співпрацювати над проєктами.

    Можна уявити Git як «машину часу» для вашого коду, яка дозволяє повертатися в минуле, переглядати історію змін та визначати, хто і коли внесе зміни. Також можна скасувати зміни, внесені раніше.

  • Термінологія

    • Репозиторій (Repository)

      Репозиторій Git — це сховище для вашого проєкту. Він може бути локальним на вашому комп'ютері або віддаленим на сервісі, такому як GitHub. Використовується для відстеження змін, координації роботи та збереження історії проєкту.

      Коли ви ініціалізуєте репозиторій Git у директорії з файлами проєкту, Git створює приховану піддиректорію .git, де зберігається інформація про історію та поточний стан. За замовчуванням ця директорія прихована.

    • Коміт (Commit)

      Коміт в Git — це знімок стану вашого проєкту в конкретний момент часу. Він містить інформацію лише про зміни, внесені після останнього коміту, утворюючи ланцюжок змін у проєкті.

    • Гілка (Branch)

      Гілка — це альтернативна версія репозиторію, що дозволяє працювати над конкретною функцією, не впливаючи на основний проєкт. Після завершення роботи гілку можна об'єднати з основною версією.

      У репозиторії завжди є принаймні одна гілка, часто названа 'main' або 'master'.
